## 第三部分：测试场景描述（核心输入）

> 用自然语言写操作步骤，每步后写校验。
>
> 工具能自动处理的操作：打开页面、点击按钮、填写字段、下拉选择、搜索、保存、发布、删除。
> 工具能自动处理的校验：页面无错误、数据库有记录/状态正确、列表显示内容、异步轮询等待。

### 编写格式

```
1. [操作步骤]
   -> 校验：[校验内容]

2. [操作步骤]
   -> 校验：[校验内容]
```

字段约束用括号标注跟在字段名后：
```
- 设备名称（必填|最长50字符|不可重复）= AUTO_DEV_001
- 设备编码（字母开头+数字）= AUTO_DEV_001
- IP（0-255格式）= 10.20.30.40
```

特殊标注：
```
- （从下拉搜索选择已发布的XXX）    → 关联搜索+选择
- （注意：已发布不可编辑）         → 状态操作限制
- （预期失败：XXX）                → 预期会失败的操作
- 依赖声明：本脚本依赖「XXX」已发布 → 前置数据依赖
- 最多等待 30 秒                   → 异步轮询超时
```

### 完整示例：设备管理·基础功能（6场景）

```
依赖声明：本脚本依赖「设备模型」已发布 +「元件」已发布，用于设备创建时的下拉选择。
          创建顺序：设备模型（创建→发布）→ 元件（创建→发布）→ 设备

前置清理：从数据库删除所有 AUTO_DEV_ 前缀的旧测试数据（物理 DELETE），
          避免名称唯一约束冲突。

场景1：新增设备

1. 打开设备列表页（URL: /controller/cDeviceList）
   -> 校验：页面加载完成，搜索框可见

2. 点击新增按钮，填写：
   - 设备名称（必填|最长50字符|不可重复）= AUTO_DEV_设备_001
   - 设备编码（必填|字母开头+数字|不可重复）= AUTO_DEV_001
   - IP 地址（0-255格式）= 10.20.30.40
   - MAC 地址（大写字母+数字+下划线连接）= AA-BB-CC-DD-EE-FF
   - 型号 = 测试型号
   - 厂商 = 测试厂商
   - 安装位置 = 测试机柜A区
   - 设备模型（必填|从下拉搜索选择已发布的模型）= AUTO_MODEL_设备模型_001
   -> 校验：保存后页面无错误提示
   -> 校验：数据库 device 表有记录，device_name = 'AUTO_DEV_设备_001'
   -> 校验：数据库 device_status = 'draft'

3. 搜索该设备，点击编辑进入编辑页二次确认
   -> 校验：编辑页各字段值与创建时一致

场景2：搜索设备

1. 在搜索框输入设备名称「AUTO_DEV_设备_001」
   -> 校验：表格中至少出现 1 行数据，名称匹配

2. 清空搜索框，输入不存在的名称「AUTO_DEV_NONEXIST」
   -> 校验：表格显示"暂无数据"

3. 按设备编码搜索「AUTO_DEV_001」
   -> 校验：搜索结果精确匹配该编码

场景3：查看设备详情

1. 搜索设备「AUTO_DEV_设备_001」
2. 点击该行的「查看详情」按钮
   -> 校验：详情页加载成功，显示设备名称

场景4：编辑设备

注意：编辑操作只能在草稿（draft）状态的设备上执行。已发布设备编辑按钮禁用。

1. 搜索设备「AUTO_DEV_设备_001」
2. 点击「编辑」按钮，修改：
   - 设备名称 = AUTO_DEV_设备_001_已编辑
   - 安装位置 = 测试机柜B区
   -> 校验：保存后页面无错误提示
   -> 校验：数据库 device_name 已更新，device_status 仍为 'draft'

3. 再次搜索并进入编辑页
   -> 校验：编辑页显示修改后的值

场景5：发布设备

注意：发布是异步操作，需轮询等待状态收敛。发布后不可编辑和删除。

1. 搜索设备「AUTO_DEV_设备_001_已编辑」
2. 点击该行的「发布」按钮
   -> 校验：弹出确认弹窗，文案包含"确认发布"

3. 点击弹窗中的「确定」
   -> 校验：最多等待 30 秒，每 5 秒检查一次状态列
   -> 校验：列表状态列变为"发布"
   -> 校验：数据库 device_status = 'release'
   -> 校验：编辑按钮已不可点击

场景6：删除设备

注意：已发布设备不可删除。需先新建一个草稿设备用于删除测试。
      删除是软删除（is_delete = true），记录保留在 DB 但列表不再显示。

前置准备：按场景1相同的规则创建草稿设备「AUTO_DEV_待删除_001」

1. 搜索草稿设备「AUTO_DEV_待删除_001」
2. 点击该行的「删除」按钮
   -> 校验：弹出确认弹窗，文案包含"确认删除"
3. 点击弹窗中的「确定」
   -> 校验：等待 2 秒等 Vue DOM 更新，重新搜索
   -> 校验：搜索结果为空（已从列表移除）
   -> 校验：数据库该记录 is_delete = true
```

## 使用方式

填完后发给 AI Agent，说：

> **「启动 web-auto-pipeline，按此模板执行」**

工具会自动完成：
1. Phase 0：自动登录、扫描菜单、探索每页路由和组件、识别表单字段
2. Phase 1-2：按场景描述自动生成脚本、执行、截图、断言、生成 HTML 报告
3. Phase 3-4：聚合报告、故障分析

你不需要填菜单结构、URL 路由、组件类型、定位方式——这些都是工具自己探索出来的。
