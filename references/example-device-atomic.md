## 第三部分：场景描述

```
依赖声明：本脚本依赖「设备模型」已发布 +「元件」已发布，用于设备创建时的下拉选择。

前置清理：从数据库 device 表物理 DELETE 所有 AUTO_DEV_ 前缀的旧数据。
          同步清理 device_relation_ship 关联记录。

场景1：新增设备

1. 打开设备列表页（URL: /controller/cDeviceList）
   -> 校验：页面加载完成，搜索框可见

2. 点击新增按钮，填写：
   - 设备名称（必填|最长50字符|不可重复）= AUTO_DEV_设备_001
   - 设备编码（必填|字母开头+数字|不可重复）= AUTO_DEV_001
   - IP 地址（0-255格式）= 10.20.30.40
   - MAC 地址（大写字母+数字+下划线连接）= AA-BB-CC-DD-EE-FF
   - 型号 = 测试型号
   - 厂商 = 测试厂商
   - 安装位置 = 测试机柜A区
   - 设备模型（必填|从下拉搜索选择已发布的模型）= AUTO_MODEL_设备模型_001
   -> 校验：保存后页面无错误提示
   -> 校验：数据库 device 表有记录，device_name = 'AUTO_DEV_设备_001'
   -> 校验：数据库 device_status = 'draft'

3. 搜索该设备，点击编辑进入编辑页二次确认
   -> 校验：编辑页各字段值与创建时一致

场景2：搜索设备

1. 正向搜索：输入设备名称「AUTO_DEV_设备_001」
   -> 校验：表格至少出现 1 行，名称匹配
2. 负向搜索：输入不存在的名称「AUTO_DEV_NONEXIST」
   -> 校验：表格显示"暂无数据"
3. 编码精确搜索：输入「AUTO_DEV_001」
   -> 校验：搜索结果精确匹配该编码

场景3：查看设备详情

1. 搜索设备「AUTO_DEV_设备_001」
2. 点击「查看详情」按钮
   -> 校验：详情页加载成功，显示设备名称

场景4：编辑设备

注意：只能在草稿（draft）状态的设备上执行编辑。

1. 搜索设备「AUTO_DEV_设备_001」
2. 点击「编辑」，修改：
   - 设备名称 = AUTO_DEV_设备_001_已编辑
   - 安装位置 = 测试机柜B区
   -> 校验：保存后页面无错误
   -> 校验：数据库 device_name 已更新，device_status 仍为 'draft'
3. 再次搜索并进入编辑页
   -> 校验：编辑页显示修改后的值

场景5：发布设备

注意：发布是异步操作，需轮询等待。发布后不可编辑和删除。

1. 搜索设备「AUTO_DEV_设备_001_已编辑」
2. 点击「发布」按钮
   -> 校验：弹出确认弹窗，文案包含"确认发布"
3. 点击弹窗中的「确定」
   -> 校验：最多等待 30 秒，每 5 秒检查一次状态列
   -> 校验：列表状态列变为"发布"
   -> 校验：数据库 device_status = 'release'
   -> 校验：编辑按钮已不可点击

场景6：删除设备

注意：已发布设备不可删除。需新建草稿设备用于删除测试。删除是软删除。

前置准备：按场景1规则创建草稿设备「AUTO_DEV_待删除_001」

1. 搜索「AUTO_DEV_待删除_001」
2. 点击「删除」按钮
   -> 校验：弹出确认弹窗，文案包含"确认删除"
3. 点击「确定」
   -> 校验：等待 2 秒，重新搜索
   -> 校验：搜索结果为空
   -> 校验：数据库该记录 is_delete = true

脚本结束前清理所有 AUTO_DEV_ 前缀的数据。
```
