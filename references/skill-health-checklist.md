# Skill 健康分析检查清单

> 应用于 web-auto-pipeline 的优化会话总结。适用于任何 Hermes Skill 的结构一致性审查。

## 审查维度（6 项）

### 1. 交叉引用完整性

- [ ] SKILL.md 每个 `references/xxx.md` 引用指向实际存在的文件
- [ ] SKILL.md 中的节号引用（如 §10）有对应的实际节号
- [ ] core-principles 等外部文件的标题声明与 SKILL.md 引用一致
- [ ] 所有文件路径引用（绝对路径和相对路径）指向真实文件

### 2. 规则/断言一致性

- [ ] SKILL.md MUST 级规则与 reference 文件中的实现代码不矛盾
- [ ] 选择器范围检查：`.el-message` bare 会匹配 `.el-message--success` 造成假阳性
- [ ] 函数签名一致性：`log()`, `check_page_errors()`, `fill()`, `ss()` 等在不同文件中的定义一致
- [ ] "已过时"标记的文件必须有醒目的替代路径说明

### 3. 幽灵文件检测

- [ ] 统计 references/ 目录中的总文件数
- [ ] 对比 SKILL.md 文件索引表中明确列出的文件
- [ ] 区分三种状态：已索引、引用但未索引、完全无人引用
- [ ] 完全无人引用的文件按处置策略处理（合并→索引→删除）

### 4. 重复内容检测

- [ ] 同一段代码/流程图是否在多个文件中重复出现
- [ ] 索引行数与实际文件行数差值 >10 行以上
- [ ] 重复的节号标题或 section

### 5. 结构一致性

- [ ] 编号顺序是否正确（如 §11.0 → §11.0a → §11.0b 不能是 b 在 a 前）
- [ ] 规则标题与规则实体数量匹配（"10 条"但实际有 11 条）
- [ ] 分隔线 `---` 无重复残留

### 6. 运行时验证

- [ ] `run.py --list-scripts` 可正常执行
- [ ] `core/` 下所有 `.py` 文件可 AST 解析
- [ ] `platforms/` 下所有脚本无语法错误
- [ ] `from config import ...` 导出的变量数量与文档一致

## P0/P1/P2 严重性分类

| 级别 | 定义 | 示例 |
|:---|:---|:---|
| **P0** | 会导致 agent 静默产生错误结果或循环引用 | selector 匹配到 success 消息导致假阳性、过时结论"不可自动化"误导 agent |
| **P1** | 会导致 agent 困惑、找不到资源、做出次优判断 | 死链接、交叉引用指向不存在的节、文件路径指向错误位置 |
| **P2** | 结构性问题，不影响当前执行但反映设计欠债 | 过时计数、重复分隔线、孤立文件未注册 |

## 修复优先级原则

1. P0 当场修复，不讨论不等待
2. P1 从专业角度给最优方案，直接执行
3. P2 一并修复，不再单独征求意见
4. 破坏性操作（删除文件、合并覆盖）前预览文件清单
